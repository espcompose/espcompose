### Array globals and useRetainedGlobal

**@espcompose/core**

- Added array global support (`'integer[]'`, `'float[]'`, `'boolean[]'`, `'string[]'`) to `useGlobal()`, backed by `std::vector<T>`. Array handles expose `.get(i)`, `.set(i, v)`, `.push(v)`, `.clear()`, and `.length`.
- Introduced `useRetainedGlobal()` hook for flash-persistent globals with an explicit stable key, separating volatile and retained global semantics.
- Simplified `useGlobal()` API to accept TS-native type tokens (`'integer'`, `'float'`, `'boolean'`, `'string'`) instead of raw C++ type strings. Keys are now auto-generated by the compiler.
- Added `IRExprArrayIndex`, `IRExprArrayMethod`, and `ArrayMethod` to the expression IR, and `IRArraySet`, `IRArrayPush`, `IRArrayClear` to the action IR.
- Added array ExprType variants (`int_array`, `float_array`, `bool_array`, `string_array`).
- Fixed reactive `padRow`/`padColumn` layout values being passed directly to LVGL Python codegen (which doesn't accept lambdas) — they are now promoted to top-level reactive bindings.

**@espcompose/cli**

- Updated the reactive transformer to inject `__key` into volatile `useGlobal()` calls via a new `global-key-injector` pass.
- Extended the expr compiler to recognise array handle `.get(i)` and `.length` accesses, emitting `array_index` and `array_method` IR nodes.
- Extended the action compiler to handle `.set(i, v)`, `.push(v)`, and `.clear()` on array globals, emitting the corresponding array action IR.
- Added `useRetainedGlobal()` scanning to both the expr compiler and script transformer.
- Extracted shared `buildScriptCtxWithGlobals()` helper to avoid duplicated global-handle-to-script-context logic in action and condition compilation.

**@espcompose/ui**

- `Space` component now spreads all `style` props onto the underlying `lvgl-obj`, preserving user-provided styles that were previously being dropped.
